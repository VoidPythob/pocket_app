# 塔语构词规范

本规范以 [docs/hymmnos-words.md](../../docs/hymmnos-words.md) 中已有词条为依据，只做“词表归纳式”构词总结。
它适合拿来约束新造词的风格一致性，但不宣称等同于官方完整语法。

## 使用范围

- 适用于普通词汇、诗性词汇、术式词汇的命名与扩写。
- 优先保证“像现有塔语词表里会出现的词”，其次才追求机械一致。
- 当词表证据不足时，优先沿用现有词根复合，不凭空发明大量新后缀。

## 词条层级

词表中大致可分为三层：

- 普通词：`n.` `v.` `adj.` `adv.` `conj.` `prep.` `prt.` 等，可自由参与复合。
- 情感句式：`E.S. (I/II/III)`，更像固定情绪表达，不应当视为普通词根随意拆拼。
- 术式/调用式：`E.V.` 与 `cnstr.`，多用于咏唱、系统调用、构造名，形式上与普通词分层。

因此，造词时先判断你要造的是：

- 日常词汇
- 情感表达
- 术式/构造名

三者不要混写。

## 正字与外形规范

- 普通词默认使用小写拉丁字母，如 `fau` `dauan` `fernia`。
- 特殊构造名允许大写、驼峰或下划线，如 `ChronicleKey`、`D_CELL`、`EXA_PICO`。
- 术式缩略式允许点号分段，如 `c.z.`、`d.n.`、`f.w.r.`。
- 普通复合词默认直接连写，不加空格、不加连字符，如 `dornpica`、`bengnuih`、`frawrle`。

## 核心构词原则

### 1. 先词根，后扩义

新词优先从已有词根出发，再做语义扩展。
不要先想中文概念，再硬拼陌生字母串。

推荐顺序：

1. 找主语义词根。
2. 判断它是名词、动作还是性质。
3. 再决定是加派生、做复合，还是转入术式层。

例如词表里已有：

- `fau` = bird
- `dauan` = dawn
- `colga` = ice
- `frawr` = flower

所以新词更应该沿这些根扩，而不是跳开另造完全无关的根。

### 2. 复合优先于生造后缀

词表中最稳定的现象不是复杂屈折，而是“紧密复合”。
能用两个已有成分表达的，优先直接复合。

现有例子：

- `dornpica`：由前项与后项连写形成具体事物名。
- `bengnuih`：表现时间语义叠接。
- `frawrle`：由基础名词扩展为相关时令/领域词。

规范上建议：

- 一个词里保留 1 个主根，最多再接 1 到 2 个补充成分。
- 复合后允许意义发生轻微诗性偏移，不要求逐字直译。
- 如果复合后已经能表达概念，不再额外叠加装饰性后缀。

### 3. 功能词独立，不黏附成词尾

`ag`、`en`、`den`、`anw`、`du`、`dn`、`ess`、`dea` 这类连接词、介词、标记词在词表中是独立词。
构词时不要把它们机械黏到普通词尾部伪装成新词。

也就是说：

- 句法关系尽量靠独立词表达。
- 词汇扩义尽量靠词根复合表达。

## 可归纳的派生模式

以下模式分为“高置信”和“暂定可用”两类。

### 高置信模式

#### 1. `-e`：关系化、属性化

词表中最清楚的例子是：

- `dauan` = dawn
- `dauane` = like or related to dawn

因此可以归纳：

- 当基础词为名词时，`-e` 可用于生成“与其相关的、具有其性质的”形容词。

推荐语义：

- 领域相关
- 气质相关
- 意象相关

推荐写法：

- `X` -> `Xe`

适用场景：

- 名词转形容词
- 意象词转修饰词

不建议：

- 把 `-e` 当成所有形容词的通用尾巴滥用
- 在已经明显是形容词的词根后继续叠加 `-e`

### 暂定可用模式

#### 2. `-ra`：小型化、个体化、派生实体

词表中的直接例子：

- `fau` = bird
- `faura` = small bird

可作保守归纳：

- `-ra` 可以把一个基础实体词变成更小、更轻、更具体的个体词。

推荐只用于：

- 生物
- 小型实体
- 柔和或亲昵称呼

由于词表证据还不算多，不建议把 `-ra` 当成普适“小词尾”。

#### 3. 状态/变化派生

词表可见：

- `colga` = ice
- `colgen` = freeze / frozen
- `chs` = become
- `chsee` = transform into

这说明塔语里存在“从实体到状态”“从变化到结果”的派生习惯，但具体后缀并不统一。

因此规范建议：

- 若要表达“变成 X / 进入 X 状态”，优先查现有变化动词是否可复用。
- 若必须造新词，优先造“X + 变化词”的复合表达。
- 不把 `-gen`、`-ee` 写成完全通用的语法后缀。

换句话说：

- `colgen`、`chsee` 更适合作为词族线索
- 不适合作为百分百可套模板的硬规则

## 词类扩展建议

### 名词

适合表示：

- 实体：`fau` `frawr` `dorn`
- 时间：`dyya` `adyya` `fedyya`
- 抽象概念：`arhou` `enesse` `erphy`

造新名词时优先：

- 基础根词
- 根词复合
- 由现有形容词或动词语义实体化

### 形容词

词表中的形容词来源较杂，不只有一个固定尾缀：

- 基础形容词：`briyua` `clyncye` `engua`
- 名形兼类：`burle` `dazua` `dius`
- 名词派生属性词：`dauane`

因此形容词不要求统一尾形，但应遵守：

- 能直接用现成形容词就不用生造
- 从名词转属性词时优先考虑 `-e`
- 保持音节自然，不为对称强行加尾巴

### 动词

词表中很多动词本身就是独立词：

- `arsye`
- `aterra`
- `aulla`
- `echrra`
- `fernia`

所以动词构词建议是：

- 优先独立造根或复用现有动词
- “使成为 / 变为 / 进入状态”优先与变化类词组合
- 不强行把所有名词都规则化动词化

## 情感层与术式层的专门规则

### E.S. 词

`E.S.` 更像整句情感块，而不是普通词根。
例如：

- `au`
- `apea`
- `erra`
- `ga`
- `gagis`

规范要求：

- 不把 `E.S.` 拆开当普通前后缀使用。
- 新造情感表达时，优先仿照它们的“整块句义”风格，而不是名词加形容词拼接。
- 情感等级 `(I/II/III)` 可理解为强度或句式层级，新增时要保持层次一致，不要混标。

### E.V. 与 cnstr.

`E.V.` 和 `cnstr.` 明显偏向术式、调用、系统构造：

- `c.z.`
- `d.n.`
- `f.w.r.`
- `addressADDR`
- `D_CELL`

规范要求：

- 这类词不参与普通词汇复合。
- 新造术式名时可使用缩略、点分、全大写或构造名格式。
- 术式命名优先体现功能，不追求日常词那种自然语感。

## 造词流程

### 普通词造词流程

1. 先确认目标概念属于实体、动作、性质还是抽象概念。
2. 从词表中找最接近的主词根。
3. 优先判断能否直接复合。
4. 只有在“相关属性化”时再优先考虑 `-e`。
5. 只有在“小型个体化”时谨慎考虑 `-ra`。
6. 最后检查是否仍保持塔语已有的音形风格。

### 术式词造词流程

1. 先定义功能。
2. 决定是缩略术式、构造名还是系统标识。
3. 与普通词层分离书写。
4. 必要时允许大写、点号、下划线。

## 禁止事项

- 不要把中文或英文逐字直译成过长串联词。
- 不要把所有新词都套成同一个后缀风格。
- 不要把 `prep.` `conj.` `prt.` 当成万能构词材料。
- 不要把 `E.S.` 与 `E.V.` 当普通词根拆开拼接。
- 不要为了“像异世界语言”而堆砌大写、点号和符号。

## 推荐输出格式

当后续根据本规范输出新词时，建议统一用下面格式：

```md
- 词形：
- 词类：
- 构词：
- 字面义：
- 引申义：
- 依据：
```

示例：

```md
- 词形：dauane
- 词类：adj.
- 构词：dauan + e
- 字面义：与黎明相关的
- 引申义：晨曦般的、破晓气质的
- 依据：词表中存在 `dauan -> dauane` 的关系化模式
```

## 最终原则

塔语构词最重要的不是“后缀表做得多完整”，而是三点：

- 保留已有词根的诗性语义
- 复合自然，层级分明
- 普通词、情感句式、术式词三套系统不要混用

如果词表证据不足，宁可少造、稳造，也不要写出看似规则整齐但明显脱离 `hymmnos-words` 风格的新词。
